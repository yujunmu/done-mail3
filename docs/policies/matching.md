# 匹配规则

匹配规则用于判断一封邮件是否命中策略。

## 条件模式

| 模式 | 说明 |
| --- | --- |
| 全部条件命中 | 所有条件都成立时才命中 |
| 任一条件命中 | 任意一个条件成立即命中 |

没有添加条件时，策略会匹配全部邮件。

## 匹配字段

| 字段 | 说明 |
| --- | --- |
| 发件人 | 发件人名称和发件邮箱组合后的文本 |
| 发件域名 | 发件邮箱的域名 |
| 主题 | 邮件主题 |
| 正文 | 纯文本正文、HTML 正文和正文摘要 |
| 收件邮箱 | 邮件归属的收件邮箱 |
| 收件域名 | 收件邮箱的域名 |
| 是否有附件 | 使用 `true` 或 `false` 匹配 |

## 操作符

| 操作符 | 说明 |
| --- | --- |
| 包含 | 字段内容包含配置值 |
| 等于 | 字段内容等于配置值 |
| 开头是 | 字段内容以配置值开头 |
| 结尾是 | 字段内容以配置值结尾 |

匹配不区分大小写。

## 性能建议

正文匹配会读取完整正文，开销高于发件人、发件域名、收件邮箱、收件域名和主题。

高频策略建议优先使用轻量字段；只有确实需要判断正文内容时，再使用正文匹配。
